# PR intent — note-task-split

노트 (메모/일기) / 작업 (일정/할일) 두 도메인으로 환원. 메모 안 체크박스 통증 해결 — 자동 sync 폐기하고 명시적 일방향 복사로.

**문제 (dogfood 통증)**:
1. 메모 안 `- [ ]` 자동 todo 등장 → 회의록 의안 / 타인 할일 / 옵션 list 까지 false positive
2. 메모 라인 본문 수정 시 todo 페이지에서 박은 메타 (date/category) 같이 사라짐 — 라인 안 inline 토큰이라 라인 통째 replace
3. 일정 = Schedule entity 가 todo view 뿐인데 UI/라벨/입력 모달 분리. mental model 분열
4. 캘린더 셀에 같은 task 가 schedule 줄 + todo 줄 두 번 등장 (`CalendarPage.tsx:107-125`)

**해결**: Note (md 파일, 폴더 분리) / Task (inbox.md 안 `- [ ]`) 두 모델만. 메모 → Task 는 명시적 ⌘⏎ + TaskAddModal prefill (Apple Notes → Reminders 패턴). 메모 라인은 그대로 (일방향, 독립).

작업:
- journals schema meetings 와 통일 (lazy uid migration)
- Task scan 범위 축소 (inbox.md only) + Schedule entity / `_is_event` 폐기 + 캘린더 중복 push fix
- TaskAddModal 통합 (`ScheduleAddModal` 리네임/확장, 날짜/시간 optional, prefill props)
- 메모 "할일로 보내기" 액션 (`⌘⏎` + hover 아이콘, `extractTodos` parser 재사용)
- "일정" 라벨 UI 카피 정리
- migration 결정 (자동 안 함, 사용자 수동)
